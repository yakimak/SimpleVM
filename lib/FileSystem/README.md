## Лабораторная работа №2 (Информатика: организация и поиск данных)

**Тема**: сравнение алгоритмов поиска при построении гистограммы распределения значений.

### 1. Техническое задание (ТЗ)

**Цель работы**  
Исследовать эффективность различных структур данных и алгоритмов поиска при решении задачи построения гистограммы для числовых данных. Сравнить время построения гистограммы и операций поиска для разных реализаций ассоциативной памяти.

**Задача обработки информации (раздел 2 задания)**  
Выбрана задача **2.1. Построение гистограммы** (тип «словарь»).  
Вход:  
- последовательность чисел (целые или вещественные значения),  
- параметры гистограммы (диапазон значений, количество интервалов или фиксированная ширина интервала).  
Выход:  
- ассоциативная структура `ключ → количество`, где ключ — интервал (диапазон значений), значение — число элементов, попавших в этот интервал.

**Реализуемые уровни задачи**  
- Базовый уровень: хранение количества элементов на интервал.  
- Базовый+ уровень: параметризация критерия — поддержка гистограммы по различным «полям» объекта (в программе: по самому числу или по функции от числа).

**Структуры данных и алгоритмы (раздел 3 задания)**  
Реализуются следующие варианты `IDictionary<Key,Value>`:
- **SortedArrayDictionary**  
  - внутренняя структура: отсортированная последовательность пар `key–value` (динамический массив);  
  - поиск: бинарный поиск;  
  - вставка: вставка с сохранением упорядоченности (сдвиг хвоста).
- **BinarySearchTreeDictionary**  
  - внутренняя структура: несбалансированное двоичное дерево поиска;  
  - поиск: спуск по дереву;  
  - вставка: стандартная вставка в дерево поиска.
- **HashTableDictionary**  
  - внутренняя структура: хеш-таблица с открытой адресацией и линейным пробированием;  
  - хеш-функция для целых ключей;  
  - автоматическое расширение таблицы при заполнении.

Для каждой структуры реализуются операции:
- проверка наличия ключа,
- получение значения по ключу,
- добавление/обновление пары `key → value`,
- количество элементов, вместимость (для хеш-таблицы).

**Требуемый функционал программы**

- Загрузка исходных данных:
  - ручной ввод чисел с клавиатуры;
  - генерация случайной последовательности заданного размера в заданном диапазоне.
- Настройка параметров гистограммы:
  - указание минимального и максимального значений;
  - указание количества интервалов (равномерное разбиение);
  - выбор критерия (по исходному числу или по функции `f(x) = floor(x)`).
- Построение гистограммы:
  - выбор реализации словаря (SortedArray / BST / HashTable);
  - построение гистограммы на одних и тех же входных данных;
  - измерение времени построения (std::chrono).
- Сравнение алгоритмов:
  - последовательный запуск построения гистограммы для всех реализованных структур;
  - вывод сравнительной таблицы времен;
  - возможность многократного повторения измерений для сглаживания погрешности.
- Просмотр результатов:
  - текстовый вывод гистограммы на экран (интервал и количество);
  - опция сохранения результата и измерений в файл CSV для последующего построения графиков (Excel, LibreOffice Calc и т.п.).

**Тестирование**

- Встроенный тестовый режим (запуск с выбором пункта меню «Запустить тесты»), который:
  - проверяет корректность базовых операций словарей (вставка, поиск, обновление);
  - проверяет корректность построения гистограммы на небольших заранее известных наборах данных;
  - проверяет граничные случаи (пустая последовательность, один элемент, элементы на границах интервалов).

**Пользовательский интерфейс**

- Консольное меню:
  - выбор режима (ручной ввод, генерация данных, сравнение алгоритмов, тесты, выход);
  - аккуратные текстовые подсказки и проверка корректности ввода;
  - отображение исходных данных (по запросу) и результатов.

**Ограничения и допущения**

- Реализация ориентирована на одномерные числовые данные (тип `double`).
- Интервалы гистограммы равномерные по ширине.
- Число элементов может быть до нескольких миллионов (ограничено объёмом ОЗУ и временем работы).

### 2. Сборка и запуск

Требуется компилятор C++17 и CMake.

```bash
cd "2_lab"
cmake -S . -B build
cmake --build build
./build/lab2
```

### 3. Структура проекта

- `README.md` — данное техническое задание и инструкция по запуску.
- `CMakeLists.txt` — файл конфигурации сборки.
- `src/main.cpp` — точка входа, консольный интерфейс.
- `src/idictionary.h` — интерфейс `IDictionary<Key,Value>`.
- `src/sorted_array_dict.h` — реализация словаря на отсортированном массиве.
- `src/bst_dict.h` — реализация словаря на двоичном дереве поиска.
- `src/hash_table_dict.h` — реализация словаря на хеш-таблице.
- `src/histogram.h` — построение гистограммы поверх абстрактного словаря.
- `src/tests.h` — простые тесты для алгоритмов.
